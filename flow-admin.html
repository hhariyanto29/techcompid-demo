The admin console, full visibility.

11 modules covering the entire festival operation — from member database and transaction streams to daily settlements, point rules, reward catalogues, event management, and reports.

Settlement Cycle
01
Start with the overview.

Admin logs in to the Dashboard — the single pane of glass for festival operations. KPIs (members, visitors, transactions, revenue, points, redemptions, pending settlements) update in real-time.

KPIs at a glance: members, visitors, transactions, GMV, points, redemptions, pending settlement.
Open Admin Dashboard
discoverykutafestival.id/admin
02
Manage the member database.

Search, filter, inspect the full member list — every signup with demographics, transaction history, current balance. Export to CSV.

Per member: name, email, WhatsApp, country, signup date, points, spending, transactions, status.
discoverykutafestival.id/admin/members
03
Manage merchants & bank accounts.

See all UMKM merchants, categories, booth locations, registered bank accounts (where settlements go), revenue, and settlement status. Add new merchants mid-festival.

Per merchant: name, category, booth, bank account, total transactions, revenue, avg spending, settlement.
discoverykutafestival.id/admin/merchants
04
Audit every transaction.

Real-time transaction stream from all merchants. Filter by status, payment method, time range. This is the source of truth for revenue and settlement calculations.

Per transaction: TXN ID, member, merchant, amount, payment method, status, points credited, time.
discoverykutafestival.id/admin/transactions
05
Run settlements to UMKM.

Discovery Mall is the merchant of record — payment gateway funds all flow to Discovery first. Daily batch settles to each UMKM's bank account, net of MDR fees. Audit log preserves every payout.

Daily payout automatic at 09:00 WITA. Manual "Run Payout Now" available for special cases.
Open Settlements
discoverykutafestival.id/admin/settlements
06
Configure point rules.

Define how points are earned: transaction rate (Rp 10,000 = 1 pt), registration bonus, daily check-in, event attendance. Full audit log shows every point change, including manual admin adjustments.

Rules + Audit: change rules anytime; log captures every change with admin signature.
discoverykutafestival.id/admin/points
07
Curate the reward catalogue.

Add, edit, archive merchandise. Set point cost, manage stock, upload images. See popularity and adjust supply.

Per reward: name, image, point cost, stock, total redeemed, status.
discoverykutafestival.id/admin/rewards
08
Monitor redemptions.

See pending redemptions at the booth. Confirm them as redeemed. Track expired/cancelled redemptions. Booth staff use this view to confirm reward handover.

States: reserved → redeemed → or expired / cancelled. Every redemption traceable.
discoverykutafestival.id/admin/redemptions
09
Manage events & actions.

Schedule events, define points per check-in, generate check-in codes. See participant counts in real-time during events.

Each event has a QR code for staff to record member attendance.
discoverykutafestival.id/admin/events
10
Export reports.

Bulk export filtered data for offline analysis or sharing with Arta-Graha Group. Six report buckets: members, transactions, merchants, settlements, redemptions, points audit, events.

Filters apply globally: date range, event, merchant, country, format (CSV / Excel / PDF).
Open Reports
discoverykutafestival.id/admin/reports
11
Manage admin roles.

4 role tiers: Super Admin (full access), Finance/Settlement (transactions + reports), Redemption Staff (redemptions + scanner only), Merchant/Booth Staff (tenant view only).

Least-privilege: each role sees only what they need. Audit log captures every admin action.
discoverykutafestival.id/admin/roles
End of Admin Flow

Operational command center.

Discovery Mall sits at the center of every payment flow. The admin console gives full visibility into transactions, settlements, members, merchants, and points — with role-based access for finance, operations, and Arta-Graha Group reporting.
